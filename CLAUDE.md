# Never Forget

## Commit conventions

This project uses [Conventional Commits](https://www.conventionalcommits.org/). All commit messages must follow this format:

```
<type>(<optional scope>): <description>

[optional body]
```

Types: `feat`, `fix`, `refactor`, `test`, `docs`, `chore`, `perf`, `ci`, `build`, `style`

Examples:
- `feat(overlay): add multi-monitor support`
- `fix(sync): handle missing calendar permissions gracefully`
- `refactor(db): extract query builder for event filtering`
- `test(notifications): add tests for snooze expiry edge cases`
